Описание библиотеки AVR2.dll:

Теперь относительно симуляции AVR. Как я мог заметить(а может и ни я один только),
начиная с версии 8.12.SP0+ - тормозит эмуляция некоторых AVR'ок, в том числе Arduino!
Это выглядит как отсчет времени симуляции с задержкой в реальном времени.
Как я понял это не проблема кряка или еще чего-то...
Может на это влияют другие причины, такие как наличие определённых инструкций в процессоре
на вашем компьютере, может поддержка более глубокой виртуализация или еще что-то...
Для себя, я решил эту проблему заменой файла AVR2.DLL (в моделях) на файл версии 8.11.SP1.
Пока работает без проблем даже на 8.15.SP1 - в реальном времени.
Стандартный путь замены:
c:\ProgramData\Labcenter Electronics\Proteus 8 Professional\MODELS\
(конечно наверное что-то не работает как надо, не просто так новый оригинальный файл весит +1МБ,
поэтому делайте бэкап оригинального файла перед заменой)
